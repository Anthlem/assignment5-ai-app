Machine learning is a branch of artificial intelligence that allows systems to learn patterns from data and make predictions or decisions without being explicitly programmed for every rule.
Artificial intelligence is a broader field focused on creating systems that can perform tasks that normally require human intelligence.
Machine learning is a subset of artificial intelligence.

Supervised learning uses labeled data, which means the correct output is already known during training.
In supervised learning, the model learns the relationship between inputs and known outputs.
Common supervised learning tasks include classification and regression.

Classification is used when the output belongs to a category or class.
Examples of classification include spam detection, disease diagnosis, and image recognition.
A classification model predicts discrete labels such as yes or no, spam or not spam, or cat or dog.

Regression is used when the output is a continuous numeric value.
Examples of regression include house price prediction, temperature forecasting, and sales prediction.
A regression model predicts quantities rather than categories.

Unsupervised learning uses unlabeled data, meaning the data does not include known output labels.
The goal of unsupervised learning is to find hidden patterns, structures, or groupings in data.
Common unsupervised learning techniques include clustering and dimensionality reduction.

Clustering is a method of grouping similar data points together.
Customer segmentation is a common example of clustering.
Dimensionality reduction is used to reduce the number of variables while preserving important information.

Training data is the dataset used to teach a machine learning model.
Testing data is the dataset used to evaluate how well the model performs on unseen examples.
Validation data is often used during development to tune model settings before final testing.

A feature is an input variable used by the model to make predictions.
A label is the correct output value in supervised learning.
Feature engineering is the process of selecting, transforming, or creating useful input variables.

Overfitting happens when a model learns the training data too closely, including noise and irrelevant details.
An overfit model performs very well on training data but poorly on new unseen data.
Underfitting happens when a model is too simple to capture the real pattern in the data.
A good model should balance learning the signal without memorizing noise.

Generalization is the ability of a model to perform well on new unseen data.
The main goal of machine learning is not only to fit training data but to generalize well.

A model is an algorithm or mathematical system trained to make predictions or decisions.
An algorithm is a defined procedure or method used to solve a problem.
Some common machine learning algorithms include linear regression, logistic regression, decision trees, random forests, support vector machines, and neural networks.

Linear regression is a basic algorithm used for predicting continuous values.
Logistic regression is commonly used for binary classification problems.
Decision trees make predictions by splitting data into branches based on conditions.
Random forests combine multiple decision trees to improve accuracy and reduce overfitting.
Neural networks are inspired by the structure of the human brain and are especially useful for complex tasks such as image and speech recognition.

Deep learning is a subset of machine learning based on neural networks with many layers.
Deep learning is useful for handling large amounts of unstructured data such as images, audio, and text.
Deep learning often requires more data and computational power than simpler machine learning methods.

A confusion matrix is used to evaluate classification performance.
Accuracy measures the proportion of correct predictions out of all predictions.
Precision measures how many predicted positive cases were actually positive.
Recall measures how many actual positive cases were correctly identified.
F1-score balances precision and recall into a single metric.

Mean absolute error and mean squared error are common metrics for regression tasks.
Lower error values usually indicate better regression performance.

Bias refers to error caused by overly simple assumptions in the learning algorithm.
Variance refers to error caused by sensitivity to changes in the training data.
A strong model aims to balance bias and variance.

Hyperparameters are settings chosen before training begins, such as learning rate, tree depth, or number of epochs.
Model parameters are values learned during training.
Tuning hyperparameters can improve model performance.

Cross-validation is a technique used to evaluate model stability by splitting the data into multiple parts.
It helps reduce the risk of depending too much on one train-test split.

Data preprocessing is an important step in machine learning.
Preprocessing may include handling missing values, encoding categorical variables, scaling numeric features, and removing duplicates.
Good quality data often matters as much as the choice of algorithm.

Normalization and standardization are techniques used to scale data.
Normalization usually rescales values to a fixed range, often between 0 and 1.
Standardization transforms data so it has a mean of 0 and a standard deviation of 1.

Machine learning applications include recommendation systems, fraud detection, medical diagnosis, self-driving systems, language translation, and predictive analytics.
Machine learning is powerful, but its results depend heavily on the quality of data, the choice of model, and the evaluation process.

Evaluation is the process of checking how well a machine learning model or system performs.
Model evaluation compares the system output with the expected result.
Output quality checks whether the answer is accurate, relevant, and supported by the available notes.

End-to-end task success checks whether the full system works from user input to final output.
In a web app, end-to-end success means the user can enter a question, the backend can process it, and the frontend can display the answer.
A successful end-to-end test shows that the main user workflow works correctly.

An upstream component is an earlier part of the pipeline that affects the final result.
In this app, the note processing step is an upstream component.
The note processing step converts raw notes into structured JSON.
If the raw notes are poorly formatted, the processed JSON may also be weak.

A test case is an example question used to check whether the app behaves as expected.
Representative test cases are normal examples that match the intended use of the app.
Failure cases are examples where the system may give a weak, incorrect, or unsupported result.
Testing both normal cases and failure cases helps show the strengths and limits of the system.

A baseline is a simple earlier version of a system used for comparison.
A baseline comparison helps show whether an improvement actually made the system better.
If the improved version performs better than the baseline, the change is supported by evidence.

Keyword matching is a simple search method that checks whether words from the question appear in the stored notes.
Keyword matching is easy to understand and fast to run.
A weakness of basic keyword matching is that it may match common words that are not very meaningful.

Stop words are common words such as what, is, the, and how.
Stop words usually do not help much with search.
Removing stop words can make keyword matching more focused.
Stop-word filtering helps the system compare more useful words from the question with the stored notes.

Debug metadata is internal information that helps explain how the system produced an answer.
Debug metadata can include matched keywords, matched note IDs, and output examples.
Debug metadata is useful for testing, evaluation, and troubleshooting.

A deterministic system gives predictable results when the same input is used.
A deterministic matching system is easier to debug because the matching logic can be inspected directly.
Deterministic systems are useful when the task is small, fixed, and does not require creative generation.

A retrieval-first system usually searches stored content before generating or returning an answer.
A RAG system combines retrieval with a language model to answer questions using retrieved context.
RAG can be useful when there are many files, large notes, or questions that use different wording.
RAG also adds complexity because it may require embeddings, vector storage, API calls, and extra evaluation.

A tool-first system relies on functions or deterministic steps to complete the user task.
In this app, the backend matching function acts like a simple tool that searches the processed notes.
For a small fixed dataset, a tool-first deterministic matching system can be simpler and more practical than a full RAG system.
